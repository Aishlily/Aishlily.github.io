JavaScript知识梳理--基础篇

发表于 2018-01-09 | 分类于 JavaScript技术

JavaScript是ECMAScript规范的一种实现，是一种弱类型脚本语言。

原始类型

  • Boolean
  • String
  • Number
  • Null
  • Undefined

  • Symbol(ES6新定义)

    注意： 原始类型不包含Object。

值类型 vs 引用类型

  1. 值类型

    • Boolean
    • String
    • Number
    • Undefined

    • Null

      • Symbol

      在参数传递方式上，值类型是按值传递

  2. 引用类型

    • Object类的所有，例如：Array、Function

    在参数传递方式上，引用类型是共享传递

  3. 实例：

    • a是值类型，值并没有发生改变
    • b是引用类型，值发生了改变
    • 按值传递的类型，会在入栈内存中复制一份值，值类型一般不占用太多内存，按值传递能保证访问速度；共享传递的类型，是复制其引用，而不是整个复制其值，避免内存资源浪费

作用域和闭包

  1. 执行上下文
    • 代码执行前需要解析代码，解析时会先创建一个全局执行上下文环境，将代码中的变量、函数声明都保存在全局执行上下文环境中（不包括内部函数中的变量、函数），变量值保存为undefined，函数先声明好可使用。
    • 在函数执行前，会创建一个函数执行上下文环境，保存变量定义、函数声明、this、arguments
  2. this
    • this是在执行的时候才能确认，定义的时候不能确认
    • 函数由new进行调用,this指向新创建的对象，如果直接调用构造函数，this指向widows对象
    • 函数由dot(.)进行调用，this指向dot之前的对象
    • 其他情况，this指向全局对象window
    • call、apply:
    • apply 、 call 、bind 三者都是用来改变函数的this对象的指向的；
    • apply 、 call 、bind 三者第一个参数都是this要指向的对象，也就是想指定的上下文；
    • apply 、 call 、bind 三者都可以利用后续参数传参；
    • bind是返回对应函数，便于稍后调用；apply、call则是立即调用

  3. 作用域

    • ES6以前，JavaScript没有块级作用域，只有全局作用域和函数作用域，除了全局作用域，只有函数可以创建作用域
    • ES6中使用let定义块级作用域变量
    • 作用域有上下级的关系，上下级关系的确定就看函数是在哪个作用域下创建的
    • 作用域最大的用处就是隔离变量，不同作用域下同名变量不会有冲突
    • 作用域在函数定义时就已经确定了。而不是在函数调用时确定。
    • 作用域只是一个“地盘”，一个抽象的概念，其中没有变量。要通过作用域对应的执行上下文环境来获取变量的值。同一个作用域下，不同的调用会产生不同的执行上下文环境，继而产生不同的变量的值。作用域中变量的值实在执行过程中产生的确定的
    • 一个作用域下可能包含若干个上下文执行环境，上下文执行环境是函数调用时创建的，函数完成后销毁

  4. 作用域链

    • 获取变量值时，首先从当前作用域开始取值，如果没有找到，则到创建这个变量的作用域内取值，这就是作用域链

  5. 闭包

    • 函数作为返回值
    • 函数作为参数传递

    函数执行完成后，其执行上下文没有被释放

Set和Map

  1. Set类似数组，但是不允许元素重复
  2. Map类似对象，但是Map有key，key可以是任何数据类型
  3. Set 实例的属性和方法有：
    • size：获取元素数量
    • add(value):添加元素，返回Set实例本身
    • delete(value)：删除元素，返回一个布尔值，表示删除是否成功
    • has(value)：返回一个布尔值，表示该值是否是Set实例的元素
    • clear()：清除所有元素，没有返回值
  4. Set实例的遍历方法：
    • item of Set.keys()：返回键名的遍历器
    • item of Set.values()：返回键值得遍历器。不过由于Set结构没有键名，只有键值，所以keys()和values()返回结果一致
    • item of Set.entries()：返回键值对的遍历器
    • forEach()：使用回调函数遍历每个成员
  5. Map实例的属性和方法：
    • size：获取成员的数量
    • set： 设置成员key和value
    • get：获取成员属性值
    • has：判断成员是否存在
    • delete：删除成员
    • clear：清空所有
  6. Map实例的遍历方法:
    • keys()： 返回键名的遍历器
    • values()：返回键值得遍历器
    • entries()：返回所有成员的遍历器
    • forEach() ： 遍历所有成员
