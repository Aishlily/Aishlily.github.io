CSS知识梳理--基础篇

发表于 2018-01-02 | 分类于 CSS技术

选择器、伪类、伪元素

选择器权重分四级：

  1. 内联样式，如果style="xxx"，权值为1000
  2. ID选择器，如#content，权值为100
  3. 类、伪类和属性选择器，如content、:hover、[attribute]，权值为10
  4. 元素选择器和伪元素选择器，如div、p，权值为1

需要注意的是：通用选择器（*）、子选择器（>）和相邻同胞选择器（+）并不在这四个等级中，所以，他们的权值都为0. 权重值最大的选择器优先级高，相同权重的优先级遵循后定义覆盖前面定义的情况。

属性选择器：
[attr = "name"] 这种格式要求必须与属性值完全匹配，p.important 和 p[class="important"]应用到 HTML 文档时是等价的；
[attr ~= "name"] 根据属性值中的词列表的某个词进行选择；
[attr ^= "name"] 选择 attr 属性值以 “name” 开头的所有元素
[attr $= "name"] 选择 attr 属性值以 “name” 结尾的所有元素
[attr *= "name"] 选择 attr 属性值包含 “name” 子串的所有元素
[attr |= "name"] 选择 attr 属性等于 “name” 或以 “name-“ 开头的所有元素

伪类

  • 伪类定义一个元素的特殊状态，这个状态往往和用户的动态交互行为或文档的动态结构化特征有关，因此不能提前写入HTML文档中。
  • 是以:开始的一个关键词。
  • 伪类不能单独存在，它必须附加在一个选择器上。伪类将只是定义那个选择器的特殊状态。

伪元素

伪元素用来创建HTML文档语言指定之外文档树的抽象层。伪元素允许作者引用这些除此之外无法访问的信息。伪元素还提供一种方法来引用不存在于文档中的内容。

一个伪元素由2个冒号（::）加上伪元素的名字所定义。

每个选择器中只能出现一个伪元素，并且只能出现在选择器主体的后面。

伪类和伪元素的区别

  • 两者都不存在于HTML文档树中，都是为了支持依靠文档树之外的信息来进行格式化。
  • 出于历史兼容性原因，一些在CSS2.1规范之前引入的伪元素使用和伪类一样的语法标记（单冒号:）。
  • 伪类本质上和类相同，是用来选择过滤HTML元素的。只不过伪类是按动态特征过滤，而不是名称、属性或内容。
  • 伪元素本质上和元素相同，是已有元素的某个部分或新创建的补充元素。伪元素上可以应用伪类选择器来过滤。
  • 伪类可以出现在选择器的任何地方，而伪元素只能添加在最后一个简单选择器之后。

样式继承

CSS利用HTML的文档层级关系，使得属性的值能通过层级关系从上到下的传承下去。只有一部分属性具备可继承性，主要是一些文本属性：

  • 文本颜色
  • 字体（font）(family, size, style, weight)
  • 行高（line-height）

推荐阅读：CSS特异性

颜色定义方法

  • 名称：145种颜色名称 black、white、orange、yellow、blue、green、lawngreen、orchid、crimson…
  • RGB: Red、Green、Blue,每种颜色各自使用一个字节来定义取值0-255，在rgb上加伤alpha值(从0到1，十进制数)用来表示颜色的透明度
  • 十六进制：例如：#db4e44中db对应于R颜色,4e对应于G颜色，44对应于B颜色

如果不使用透明色，建议使用16进制。如果要使用透明色，使用 rgba 颜色单位。

尺寸

很多CSS属性需要使用尺寸单位：

  • font-size 定义文本大小
  • border-width 定义元素边线宽度
  • margin 定义元素之间的距离
  • left/right/top/bottom 定义元素位置或相对偏移

CSS支持多种长度单位，最常用的尺寸单位有：

  • in 英寸
  • px 像素
  • % 百分比
  • em 相对于父元素尺寸的比值，依赖于父元素的字体大小
  • rem 相对于根元素字体大小的比值
  • ex 相对于字符“x”的高度，此高度通常为字体尺寸的一半。

绝对长度度量不依赖于显示设备的绝对尺寸来定义长度，有：英寸（in）、像素（px）。绝对单位适合用于打印/印刷网络文档的应用场合。

像素可以被看作特殊的相对单位，即相对于显示器屏幕分辨率的单位。

相对长度度量相对其它为浏览器所知的单位来定义长度，有：em、rem和percentage（百分比）。相对单位适合用于屏幕显示。

display属性

  • block 将占据整行宽度
  • inline 将被视为纯文本
  • inline-block 块和内联行为的结合
  • list-item 和 block 类似，不过会多显示一个列表符号（bullet point）。
  • table，table-row 和 table-cell 和表格相关的布局行为。

visibility: hidden; 规则和 display: none; 有点类似，用于隐藏一个元素，不同之处在于被隐藏的元素仍然占据原有的渲染空间

overflow属性

  • overflow: visible 缺省，内容总是被显示
  • overflow: scroll 内容超出容器时，用滚动条
  • overflow: hidden 隐藏掉任何溢出部分的内容
  • overflow: auto 滚动条将只在内容溢出时被自动显示。

通常为元素设定固定的尺寸是为了设计上的一致体验，但由于实际网站中内容变化区间是较大的，可能会出现意外的情况，因此为固定尺寸的元素设定合适的溢出规则：overflow: hidden 或 overflow: auto 是一个良好的开发习惯。

@media

  • width:
    • min-width: 1000px 样式仅当viewport大于1000px时被应用
    • max-width: 1000px 样式仅当viewport小于1000px时被应用
  • height: min-height 和 max-height，同width
  • orientation参数用来查询viewport是否处于以下两种模式之中：
    • 横屏（landscape）: viewport的宽度大于高度
    • 竖屏（portrait）: viewport的高度大于宽度
  • resolution: min-resolution max-resolution
