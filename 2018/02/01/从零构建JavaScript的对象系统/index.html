从零构建JavaScript的对象系统

发表于 2018-02-01 | 分类于 JavaScript技术

类与继承

类是对象的定义，而对象是类的实例（Instance）。类不可直接使用，要想使用就必须在内存上生成该类的副本，这个副本就是对象。

以Java为例：

通过继承，子类可以直接从父类获得其所有的属性和方法，继承的实现机制是”复制、拷贝”。

原型与继承　

严格的讲，JavaScript中既不存在类，也不存在实例。JavaScript中的对象是通过构造函数创建的。

没有类，那么JavaScript怎么实现继承呢？

构造函数中设置prototype属性。这个属性包含一个对象（即”原型对象”），所有实例对象需要共享的属性和方法，都放在这个对象里面；不需要共享的属性和方法，就放在构造函数里面。

与类继承的”复制、拷贝”不同，原型继承的机制是”引用、关联”。JavaScript中所有的对象都是由构造函数生成的，每个对象都共同继承构造函数的原型对象中的属性和方法。

在对象内部有两种方式访问它继承的原型：

  1. obj.constructor.prototype: 每个对象都拥有一个内置属性constructor指向其构造函数

  2. obj.__proto__: 每个对象都有一个内部指针prototype，直接指向其原型，这个内部指针是不可访问的，但是有些浏览器暴露出一个__proto__属性，用于直接访问原型对象

原型链与继承

JavaScript中的对象是通过构造函数创建的，也就是说所有对象都必然继承某个原型，而原型本身也是对象，它也会继承别的原型，如此就形成了原型链。当调用对象的某个属性时，如果对象的自有属性中不存在该属性，那么JavaScript引擎就会沿着该对象的原型链向上查找，直到找到第一个匹配的属性名为止。所有原型链的源头都是Object.prototype， Object.prototype.__proto__ ===null`

对象创建方式不同，对象的prototype具体指向/对象的原型不同。

  1. 对象变量的原型对象是Object.prototype；

  2. 通过Object.create( )创建的对象，其原型是由第一个参数指定的对象；

  3. 通过构造函数创建的对象，其原型即构造函数的原型对象。

　 自定义的构造函数，它的原型默认是一个仅含constructor属性的对象。

比较特殊的是函数也有继承的原型。注意，函数不会从自身的prototype指向的原型对象中继承任何属性或方法，所有的函数都共同继承一个原型对象，那就是Function.prototype。

为什么函数也有__proto__属性？这不奇怪，因为函数也是对象（可调用的对象）。
